TIER 1 · ANSWER

Knowledge / FAQ bot

Deflects the same questions 24/7 — hours, policies, pricing, product facts — grounded in the client’s own docs. No live systems, no write-back. The cheapest tier to run and the easiest to make reliable.

data static write-back no integration low — upload docs core build RAG + guided menus cost shape LLM call only
Example tenant

BrightSmile Dental — a single-clinic practice. The bot answers “do you do X / how much / when are you open” and hands off cleanly when it isn’t sure.

02Intent Customer types a messy, real-world question. The agent reads intent — including typos — and decides the route.

Behind the glass

AGENT
Classify intent: in-scope FAQ → pricing/services. Decide route = answer from RAG (not handoff, not a tool).
GUARD
Purpose-scoped to this clinic’s knowledge. Open-domain questions (“write me a poem”) are declined or handed off — exactly what Meta’s Jan 15 2026 policy requires.
NLU
LLM normalises “hw much”→“how much”, “u”→“you”. A keyword decision tree would miss this and fall through to a dead end.
◆ DECISION · out of scope

If intent ≠ anything this bot is built for → don’t improvise. Reply “That’s outside what I can help with — connecting you to the team” and route to the agent inbox. Staying on-task is both the policy-safe and the higher-converting design.

✓Verified against competitors & existing systemsTier 1 · Jun 2026
Flow-builders break on free text
Wati, AiSensy, Interakt & Landbot answer FAQs with rigid decision trees that stall on typos and off-script phrasing. Reliable handling of unscripted questions needs the LLM-RAG route shown above. ✓ respond.io · dmchamp · 2026
Open-domain bots are now banned
From Jan 15 2026 Meta prohibits general-purpose assistant bots; only purpose-scoped bots (FAQ, booking, order-tracking, surveys) are allowed. A scoped FAQ bot + clean handoff is the compliant pattern. ⚠ Meta Business Solution Terms · digitalapplied
In-window replies are free
Since Nov 1 2024, customer-initiated service conversations inside the 24h window are free and unlimited. Tier 1’s only real variable cost is the LLM call — matching the blueprint’s cost model. ✓ Meta for Developers · §7 cost model
TIER 2 · ACT

Live-data / transactional bot

Everything in Tier 1, plus a typed tool-calling layer. The bot looks things up in real client systems and writes back — order status, booking, rescheduling, tickets — with a confirmation step before any change. It resolves real requests without a human.

data live (read + write) write-back yes integration medium — 1–2 APIs core build T1 + typed tools + state machine key rule model never touches the DB
Example tenant

Kapde & Co — a D2C clothing store. The bot answers “where’s my order”, then reschedules the delivery — a real read + a real write, end to end.

TOOL
Resolve contact by phone in the CRM, then call get_order(order_id). The model emits the request; the tool layer authenticates and executes.

◆ DECISION · sensitive action ahead?

Reading status from your own number is low-risk. For higher-risk actions (address change, refunds) the tool layer can require light verification (order ID already given, or an OTP) before proceeding.

The circuit breaker trips → the bot does not hang or hallucinate a status. It replies “I’ve logged this — our team will update you shortly” and auto-creates a ticket via create_ticket(). Graceful degradation, every time.

GUARD

◆ NEXT TIER · this is where Tier 3 begins

Chaining several tools across systems with branching + approvals — abandoned-cart recovery, pay-in-thread, campaigns — is Tier 3 (orchestration / LangGraph). Tier 2 stops at single, confirmed actions.

✓Verified against competitors & existing systemsTier 2 · Jun 2026
Order-tracking is proven on WhatsApp
Merchants on Interakt / Wati report support response times down ~70% and ~30% more carts recovered once tracking & updates run in-chat — the exact Tier 2 read-flow shown above. ✓ Interakt · Zoko case notes · 2026
Real intent + write-back is the moat
Native intent matching without exact keywords needs Interakt’s paid Haptik add-on (₹10k+/mo); flow-builders can’t reliably run unscripted multi-step actions. The typed-tool agent does both natively. ⚠ zoko · respond.io · dmchamp
Never auto-execute money/state moves
Across the market, AI does not place orders, refund, or cancel on its own — a human stays in the loop. The Confirm gate (stage 04) is the standard, not an extra. ✓ go4whatsup · Alibaba Cloud · §4.3

Reading this as a build spec

Both journeys run on one multi-tenant platform; a tenant’s tier is a config flag that switches tools on, not a separate codebase — so Tier 1’s five stages are literally the first five of Tier 2. The left column is the only thing the customer ever sees; the right column is where your reliability, cost control, and guardrails actually live. Every number here is a placeholder with a source — Meta adjusts rates every 6–12 months, so re-verify quarterly.
