How to read this as a build spec

This is one console for a multi-tenant platform; a client’s tier is a flag that lights up modules, not a separate product. Part A is the setup wizard that earns the “Setup / onboarding” fee; Part B is the recurring surface that earns the retainer. Every screen here is the user-facing edge of a layer you’ve already specced in the blueprint — the table on the right of each screen names exactly which one. Build the shell + tier-flag system once, and every screen becomes tier-aware for free.

BLUEPRINT MAP —
Console = Layer 8 · Control plane (§4.1) · apps/console/ (§4.4)
